다음 사용자 요청을 직접 처리하지 말고, **반드시 Agent 툴로 서브에이전트에게 위임**하라.

## 라우팅 규칙

요청 내용을 보고 다음 중 하나를 선택:

### `ocr-improver` 로 보낼 키워드
- OCR / 인식률 / 정확도 / 정확하게
- 곡면 / 휘어진 / 라벨
- Tesseract / OpenCV / 전처리 / threshold / contour / 검출
- LOT / 유효기간 / 카테고리 / 분류
- Google Vision / Azure OCR / PaddleOCR / 외부 API
- 사진이 / 이미지가 / 안 잡혀 / 잘못 잡혀

### `refactor-improver` 로 보낼 키워드
- 리팩토링 / 정리 / 깔끔 / 구조 / 가독성
- 중복 / 함수 분리 / 네이밍 / 죽은 코드
- 파일 분리 / 모듈 / 빌드

### 애매한 경우
두 영역이 섞여 있으면 사용자에게 짧게 어느 쪽인지 확인. 추측해서 잘못 보내지 말 것.

### 매칭 안 되는 경우
서브에이전트로 보내기에 적절하지 않은 일(예: 단순 질문, 파일 보내기, 상태 확인)이면 위임하지 말고 직접 응답하라.

## 위임 방식

선택한 서브에이전트에 Agent 툴로 호출. 프롬프트는 다음 정보를 포함:
- 사용자의 원본 요청 (그대로)
- 현재 작업 중인 파일/줄 (대화 맥락에서 파악 가능하면)
- 기대 산출물 (조사만? 코드 변경까지?)
