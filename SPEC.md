# OCR 추출 스펙 (Coralis Reverse Shoulder System)

> 한 장의 사진에 여러 의료 임플란트 박스 스티커가 함께 찍힘.
> 스티커마다 아래 정보를 뽑아 카드 1장씩 만드는 것이 목표.

## 입력
- 사용자가 업로드한 사진 1장 (모바일/데스크탑 카메라)
- 한 사진에 보통 5~12장의 스티커가 모여 있음
- 일부 스티커는 원통형 포장에 둘러붙어 글자가 휘어 있음

## 한 스티커에서 추출할 것

### 필수 (반드시 뽑기)

#### 1. LOT 번호 (배치번호)
- 라벨에 **`LOT`** 글자 옆/아래에 위치
- 코랄리스 LOT 패턴 (휴리스틱):
  - 길이 **6~10자**
  - **숫자 2자리로 시작** (보통 `10` 또는 `1?`)
  - **영문자 2자 이상 포함**
  - 영숫자(대문자)만 사용 — 특수문자 없음
  - 순수 숫자(예: 날짜)는 LOT 아님
- 예시: `10BBBFC4`, `10BEKSCA`, `10BFCP02`
- "LOT" 글자 없이도 위 패턴이 단독으로 인쇄된 경우가 있음 → 패턴으로도 잡기
- OCR 흔한 오인식: `O` ↔ `0`, `I` ↔ `1`, `S` ↔ `5` (LOT 추출 시 `O` → `0` 변환 권장)

#### 2. 유효기간 (Expiration Date)
- 보통 **모래시계 ⏳ 아이콘** 또는 **`EXP`** 텍스트 옆에 위치
- 같은 스티커에 **여러 날짜가 인쇄**됨 (제조일 + 유효기간)
  - → **가장 미래의 날짜를 유효기간**으로 선택
- 형식 (어떤 구분자든 가능):
  - `YYYY-MM-DD`, `YYYY.MM.DD`, `YYYY/MM/DD`, `YYYYMMDD`, `YYYY MM DD`
- 유효 범위: 년 2000~2099, 월 1~12, 일 1~31
- 출력은 `YYYYMMDD` 8자리로 통일

### 분류 (선택)

#### 3. 제품 카테고리
제품명에 아래 키워드가 있으면 그 카테고리로 분류:

| idx | 카테고리 | 키워드 |
|---:|----------|--------|
| 0 | Stem | `stem` |
| 1 | Tray (Cup) | `tray` / `cup` / `humeral cup` |
| 2 | Liner (Insert) | `liner` / `insert` |
| 3 | Glenosphere | `glenosphere` (OCR로 `gienosphere` 자주 옴) |
| 4 | Baseplate | `baseplate` (OCR로 `baseplale` 자주 옴) |
| 5 | Central Screw | `central screw` |
| 6 | Locking Screw | `locking screw` **단, `non-locking` 제외** |
| 7+ | 사용자 정의 (Spacer 등) | 사용자가 동적으로 등록 |

- 매칭되는 키워드가 없으면 `categoryIdx = -1` (미식별) → UI에서 ⚠ 표시

## 스티커 전형 레이아웃

```
┌──────────────────────────────────────┐
│  [coralis 로고]              [CE 마크]│
│                                       │
│  Coralis Reverse Shoulder System      │
│  [Product Name]   ← 카테고리 키워드 위치│
│  Ø 4.5 / 36mm / Thread: 24mm          │
│                                       │
│  ║▌║│▌║║│▌ (1D 바코드)                │
│  ▓▓▓▓▓ (2D / QR 바코드)               │
│                                       │
│  REF [품번]                           │
│  LOT [번호]      ⏳ YYYY-MM-DD          │ ← 핵심
│  (또는)                                │
│  ⏳ 2025-12-31    LOT 10BEKSCA          │
└──────────────────────────────────────┘
```

위치는 스티커마다 살짝 다르지만 **`LOT` 토큰과 유효기간 토큰은 보통 같은 줄 또는 인접한 줄**.

## 출력 데이터 구조

스티커 1장 → 다음 객체:

```js
{
  lot: "10BEKSCA",            // 대문자 영숫자
  expiration: "20360228",     // YYYYMMDD
  productName: "Peripheral Locking Screw",  // 원본 OCR 텍스트
  categoryIdx: 6,             // 0~N (위 표) 또는 -1
  cropBox: {                  // OCR 캔버스 좌표 (썸네일/하이라이트용)
    x0, y0, x1, y1
  },
  recognized: true            // 카테고리 매칭 성공 여부
}
```

## 매칭 휴리스틱 (현재 구현 요약)

1. Tesseract 2-pass OCR (PSM 6 + PSM 11) → 단어 + bbox 수집
2. 모든 단어에서 LOT 후보 찾기 (`isCoralisLot` 또는 `LOT` 접두사)
3. 모든 단어/라인에서 날짜 후보 찾기 (정규식)
4. 각 LOT 후보에 대해:
   - **공간적으로 가장 가까운** 날짜들 (거리 임계 내)을 모아
   - **가장 미래의 날짜**를 유효기간으로 선택
5. LOT 근처의 위쪽 라인들에서 카테고리 키워드 찾아 productName 추출
6. 같은 LOT 값이 두 OCR pass에서 중복 발견되면 1개로 dedupe

## 알려진 어려움

| 문제 | 원인 | 임시 대응 |
|------|------|----------|
| **곡면 라벨 인식 실패** | 원통형 포장에 둘러붙어 글자가 휨 | 사용자가 ROI 드래그로 영역 지정 → 그 영역만 재 OCR |
| **`O` ↔ `0` 혼동** | OCR 한계 | LOT 추출 시 `O` → `0` 변환 |
| **2개 스티커가 한 줄에 인쇄** | 한 라인에 LOT 2개 | bbox 거리로 LOT-날짜 매칭 |
| **카테고리 키워드 부재** | 신제품 / 특수 케이스 | 사용자가 템플릿에서 항목 추가 + 키워드 등록 |
| **여러 날짜가 가까이** | 제조일 + 유효기간 | 더 미래 날짜 선택 |

## 개선 방향 (미구현)

1. **곡면 보정** — OpenCV.js 등으로 원근/곡면 변환
2. **외부 OCR API** — Google Cloud Vision, Azure OCR (곡면에 강함)
3. **바코드 디코딩** — 1D/2D 바코드 자체에서 LOT/UDI 추출 (가장 정확)
4. **카테고리 키워드 사전 확장** — 실제 카탈로그 기반
